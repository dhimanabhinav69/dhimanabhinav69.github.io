slide 01 / executive summary

the story

From PostgreSQL tables to a Tableau business dashboard.

OmniCommerce needed a simple reporting layer to understand sales performance across products, customers, regions, categories, shipping modes, and order fulfilment. I built the database first, verified the metrics using SQL, then connected the model to Tableau for an interactive dashboard story.

project objective

What this report answers

Which categories and subcategories drive revenue? Which regions and states perform best? How do customers compare by sales and profit? Which products are most profitable? How does profit vary by category and fulfilment pattern?

  • Design a 7-table relational retail database in PostgreSQL.
  • Validate Tableau numbers with SQL queries for every chart.
  • Use Tableau filters, parameters, maps, heatmaps, scatter plots, packed bubbles, and a box-and-whisker plot.

slide 05 / business problem

The stakeholder problem

Retail data usually sits across separate customer, product, order, shipment and target files. Without a properly joined and validated model, stakeholders cannot trust whether Tableau numbers match the database calculations.

  • Which categories and subcategories are driving the highest sales?
  • Which regions, segments and states should be prioritized?
  • Which customers produce high sales but weaker profit?
  • Which products appear in the Top N profit and sales list?
  • How do fulfilment timelines and shipping modes vary across orders?

slide 06 / PostgreSQL data model

Seven-table relational schema

The database is structured around orders and order items, with lookup tables for customers, products, sales representatives, shipments and monthly targets.

customersCustomer name, segment, city, state, country, region and signup date.
productsCategory, subcategory, product name, unit price and cost price.
sales_repsSales representative, region and team assignment.
ordersOrder date, customer, rep, payment mode, order status and discount percentage.
order_itemsOrder-level product quantity and selling price for line-item calculations.
shipmentsShip mode, ship date, delivery date and shipping cost for fulfilment analysis.
monthly_targetsOptional target table for region/category target comparison and SQL concept coverage.

-- KPI verification used before building Tableau cards SELECT ROUND(SUM(line_sales), 2) AS total_sales, ROUND(SUM(line_profit), 2) AS total_profit, COUNT(DISTINCT order_id) AS total_orders, ROUND(SUM(line_profit) / NULLIF(SUM(line_sales), 0) * 100, 2) AS profit_margin_pct FROM ( SELECT o.order_id, oi.quantity * oi.selling_price * (1 - o.discount_pct) AS line_sales, oi.quantity * oi.selling_price * (1 - o.discount_pct) - oi.quantity * p.cost_price - s.shipping_cost / COUNT(*) OVER (PARTITION BY o.order_id) AS line_profit FROM orders o JOIN order_items oi ON o.order_id = oi.order_id JOIN products p ON oi.product_id = p.product_id JOIN shipments s ON o.order_id = s.order_id WHERE o.order_status = 'Completed' ) x;

A clean pipeline from raw CSVs to validated Tableau visuals.

slide 10 / recommendations

Business actions suggested by the analysis.

Protect top subcategories

Focus stock planning and promotions around the strongest subcategories shown in the ranked bar chart and packed-bubble view.

Review low-margin customers

Use the scatter plot to investigate customers with large sales but comparatively weak profit contribution.

Track fulfilment delays

Use the fulfilment timeline with ship mode filters to identify orders that take longer from order date to delivery date.

Use SQL as the audit layer

Keep every Tableau KPI connected to a verification query so the dashboard remains explainable during interviews or project reviews.
